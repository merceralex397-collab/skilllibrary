# Expected Outcomes

A good Tailwind Shadcn run should:

- trigger only when the task genuinely matches the skill boundary,
- produce a response that includes `Detected Stack Signals`, `Chosen Pattern`, `Implementation Notes`, `Validation`,
- stay within the frontend workflow instead of drifting into generic advice,
- call out uncertainty and next validation when the evidence is incomplete,
- avoid fabricating implementation details or sources.
