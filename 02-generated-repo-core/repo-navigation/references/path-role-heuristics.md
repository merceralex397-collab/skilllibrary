# Path Role Heuristics

Common directory roles:

- `src`, `app`, `lib`, `server`, `client`: source
- `docs`, `specs`, `tickets`, `taskfiles`: context and planning
- `scripts`, `tooling`: operational helpers
- `test`, `tests`, `e2e`: validation
- `dist`, `build`, `coverage`, `artifacts`: derived output
- `archive`, `legacy`, `snapshots`: historical or frozen material
