# Expected Outcomes

A good Skill Catalog Curation run should:

- trigger only when the task genuinely matches the skill boundary,
- produce a response that includes `Inputs`, `Chosen Shape`, `Generated Artifacts`, `Validation and Handoff`,
- stay within the builder workflow instead of drifting into generic advice,
- call out uncertainty and next validation when the evidence is incomplete,
- avoid fabricating implementation details or sources.
