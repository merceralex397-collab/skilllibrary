# Failure Catalog

Use these as prompts, not mandatory sections.

## Product and user adoption

- solves the wrong problem
- onboarding friction kills activation
- operators refuse the new workflow

## Engineering execution

- scope expands beyond team capacity
- hidden dependency blocks implementation
- performance or reliability target is missed

## Integration and dependency

- upstream API behavior differs from expectation
- rate limits, auth, or data contracts break the plan

## Operations and observability

- rollout has no safe canary or rollback
- alerts do not fire until customers complain

## Data correctness

- migration assumptions are wrong
- source data quality is lower than expected

## Security or approval

- compliance review blocks launch late
- new logging or data flows violate policy

## Team and process

- no clear owner for the risky phase
- review and approval latency makes the schedule fake
