# Criteria Library

Use only the criteria that materially matter to the decision.

## Common criteria

- delivery speed
- implementation complexity
- operational burden
- safety / failure blast radius
- reversibility
- team familiarity
- direct cost
- long-term maintenance
- vendor or architectural lock-in
- observability and supportability

## Constraint examples

These are not weighted criteria:

- must launch before a fixed date
- must avoid downtime
- must pass a compliance requirement
- must work with the current team skill set
