# Boundary Cases

Use these prompts to expand criteria beyond the happy path.

## Common checks

- What happens when the input is empty, missing, malformed, or duplicated?
- What happens when the user lacks permission?
- What happens when the third-party dependency is slow or unavailable?
- What happens when the action is repeated?
- What happens when state changes between page load and submission?

## Do not add every edge case blindly

Choose the cases that materially change user risk, operator burden, or data correctness.
