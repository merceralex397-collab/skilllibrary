# Community Skill Audit Notes

Use community planning, scaffold, and review skills as reference material first.

Current posture:

- do not auto-install community scaffold skills into the stable core
- inspect their prompts, templates, and workflow assumptions before reuse
- extract useful patterns, then fold them into the local skill pack or scaffold assets

Notes from the initial scan:

- `project-planning` looked heavily slash-command and web-app oriented, so it was not adopted as a dependency
- `project-scaffold`, `project-workflow`, and `team-review` remain candidates for manual pattern review later
