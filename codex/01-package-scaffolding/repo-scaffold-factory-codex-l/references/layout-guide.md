# Scaffold Layout Guide

The generated scaffold is designed around a few principles:

- one obvious human handoff entrypoint
- one canonical ticket index plus per-ticket files
- one visible OpenCode leader agent with hidden specialists
- clear separation between commands, tools, plugins, skills, and MCP
- short, signposted docs that weaker models can rehydrate quickly

Core paths:

- `README.md`
- `AGENTS.md`
- `START-HERE.md`
- `docs/process/`
- `docs/spec/`
- `tickets/`
- `.opencode/`
- `opencode.jsonc`
