# Bias Checks

Watch for these failure modes in decision-making.

## Sunk-cost bias

The team prefers the option that reuses prior work even when that work no longer serves the current goal.

## Elegance bias

The team overvalues the cleaner or more intellectually satisfying design even though the operating cost is worse.

## Local-optimization bias

The option wins for one team but pushes support, ops, or migration cost somewhere else.

## False precision

The scoring table looks scientific even though the criteria or weights are arbitrary. If the inputs are weak, say so.
