## Notes on use

- For abstract skills like **steelman**, **premortem**, **plan review**, and **assumptions audit**, there usually is not a single vendor-style “official doc”. In those cases, use standards, SRE guidance, RFCs, OWASP, and NIST material as the strong support layer.
- For fast-moving categories like **MCP**, **OpenAI tooling**, **Codex**, **Claude Code**, **Vercel AI**, **vLLM**, and **Cloudflare**, prefer the product docs over third-party tutorials.
- For **LLM architecture / fine-tuning / model creation**, combine docs with **primary papers**. Docs explain how to use a stack; papers explain why the method exists and what trade-offs it introduces.
